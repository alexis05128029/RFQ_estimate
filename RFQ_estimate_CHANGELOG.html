V3.6 2026-06-24
  • FEAT
    圓棒／切割材材料計價模式：材料區可切換「每件單價 / 圓棒切割」。圓棒模式輸入「一支出數」與「整支單價（支援依支數級距）」，系統依「需求件數 = 詢價量 + 架機量」無條件進位算出需要幾支，攤提分母為需求件數。單工件與組合件皆支援，並即時顯示支數試算（如 需求 200 件 → 67 支 × NT$1000 → 每件 NT$…）。
  • FEAT
    外購件數量級距（依實際採購數量）：外購品項單價改為可設數量級距，並依實際採購數量查價——固定/件 = 訂單量 × 用量；批次攤提 = 整批用量。單工件與組合件外購料皆適用。
  • FEAT
    成本拆解雙幣別顯示：右側成本拆解每一列（含總成本）於台幣下方加顯美金（NTD ÷ 匯率），表頭標示換算匯率。
  • FEAT
    資料維護：客戶清單新增 Abbott Laboratories（FAB02）；廠外製程預設新增「外發包裝」「外發組裝」。
  • FIX
    組合件工件「料號 / 版本」欄位錯置：原本「版本」輸入存進 partNo、「料號」存進 name，導致匯出至報價單 / Excel 把版本當成料號。現改為 料號 → partNo、版本 → 新增 revision，並修正匯出、側欄拆解與 Excel；載入舊檔自動遷移欄位。
  • FIX
    組合件級距以「組數」查價：材料、廠外、外購的數量級距改用實際件數（組數 × 每組用量 N）查價，修正「每組用量 > 1」時抓錯級距、報價偏高的問題（例：N=4、50 組 → 實際 200 件 → 正確命中 ≥100 級距）。單工件不受影響。
  • CHG
    WP 合計改為「個別工件」口徑：各工件合計 = 材料（含架機攤提）+ 廠內（加工 + 機台設定費）+ 廠外；外購 / 一次性費用 / 佣金只計入總成本。對帳一致：Σ WP 合計 + 外購 + 一次性 + 佣金 = 總成本/組。
  • CHG
    每件單價模式新增計算公式提示：組合件「每件單價」模式比照圓棒模式，於材料區顯示每件材料的計算式（含架機批量攤提）。

V3.3 2026-05-27
  • FIX
    組合件模式：材料成本計算錯誤：當工件同時包含廠內加工與廠外製程時，材料成本會顯示 NT$0。原因是 calcAssemblySet() 中有一個 isOutsourced 判斷，只要工件含任何廠外製程就跳過材料成本計算。現已移除此判斷，無論製程組合為何，材料成本皆正確計算。
  • FIX
    組合件模式：Excel 匯出功能異常：在組合件模式下點選「匯出 Excel」會直接彈出「請先輸入製程資料」的錯誤提示。原因是 exportExcel() 沿用單件模式的 TABS 判斷邏輯，在組合件模式下永遠回傳空陣列。現已新增 buildAssemblyRows() 函式，並將匯出邏輯依模式分流；組合件模式會輸出含工件明細與各級距報價彙總的單一 Sheet。
  • FIX
    組合件模式：載入 JSON 後外購料清單不更新：以「載入 JSON」還原組合件模式的儲存檔後，「組合件外購料（共用）」卡片的項目清單不會重新顯示。原因是 loadCalc() 呼叫 renderPurchItems() 時未傳入 true，只更新了單件模式的外購料 UI。現已補上 renderPurchItems(true)，載入後清單可正確還原。
